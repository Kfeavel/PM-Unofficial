Project M Blog Post #4

Posted: April 14, 2011
Share Tweet Submit

Hi everyone,


Today's blogpost is focused on enhancements we have made to the game's input detection for the shoulder buttons and the C-Stick. With our former code in the current demo, any light press on a shoulder button, (whether it was to shield or air dodge, cancel a charging move, etc) would register as a full input. Our new code separates the pressing of the shoulder button into two completely separate entities. A full press still triggers as Button Press 3, and partial presses now are stored in a variable.


This has allowed for us to remove unintended action changes such as light-pressing an airdodge. This will give us that tried and true timing for wavedashing and wavelanding, and will no longer will be a throw-off in timing that many people have worried about.


An additional code also was to provide greater detail on how the C-Stick is used. The C-Stick in Brawl is actually just stored as a single button, which gives no insight as to the direction it's being held. The direction it's held is now stored in variables for X and Y components, and has enabled us to return properties back such as as true Roll/Spotdodge buffering with the C-Stick.


Stay tuned for more updates next week!
